A Excel 列别名匹配机制 — 容错导入的典范

用户导入的 Excel 列名千差万别：「编号」vs「ID」vs「序号」vs「声音 ID」，v11 通过 VOICE_COLUMN_ALIASES 做了统一的别名映射，对每个语义字段维护一组等价列名。

v11 核心代码 — 列别名匹配（行 2861-2875）
const VOICE_COLUMN_ALIASES = {
  id:      ['ID','id','声音ID','声音 ID','编号','序号'],
  model:   ['车型','型号','车辆','model','Model'],
  content: ['声音内容','内容','投诉内容','原话','content','description'],
  channel: ['渠道','来源','channel','source'],
  status:  ['状态','status','处理状态']
};

function _matchCol(headers, aliases) {
  for (const h of headers) {
    if (aliases.some(a => h && h.toString().trim() === a)) return h;
  }
  return null;
}
答辩加分点
导入时的容错设计体现了对真实用户场景的思考：不同团队用不同格式的 Excel，系统应该适配用户而非强迫用户适配系统。此外还有合并/替换选择（confirm 弹窗）、ID 冲突自动重编号（_seqAvail()）、空行跳过计数等细节。

B LLM 占位卡片模式 — 异步等待时的 UX 设计

调用 LLM 通常需要数秒等待，v11 的处理方式是先推送一张 processing 占位卡片到 DecisionStream，LLM 返回后再移除占位、插入真实结果卡片。用户不会面对空白等待。

C 防抖持久化 + 审计日志 — 企业级数据管理意识
