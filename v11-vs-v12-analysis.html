★ PocketBase API 端点（7 个自定义路由）
方法路径功能v12 复用
POST /api/llm/stage1 LLM REQ 层判断：接收声音内容 → 构造 system prompt（含模块词表 + Wiki 状态）→ 调用 DeepSeek → 返回 decision/module/reqId/confidence/reasoning/thinking。自动写 llm_calls 日志。 P0 必须
POST /api/llm/stage2 LLM TC 层 4W 比对：接收声音 + 候选 TC 列表 → 逐条 4W 字段比对 → 返回 exact/partial/fallback + newTcSuggestion。自动写 llm_calls 日志。 P0 必须
POST /api/wiki/persist 批量写入三阶段结果：exactHit → bump计数 / partialHit → 新增TC / newReq → 新增REQ+TC / crossReq → 双向crossRefs。幂等设计，可重跑。 P1 推荐
GET /api/wiki/state 知识库实时状态：totalReqs / totalVoices / totalTcs + 每个 REQ 的 voiceCount/tcCount/crossRefs 列表。 P1 推荐
GET /api/llm/stats LLM 累计统计：calls/success/failure/totalTokens/avgElapsedMs/estCostYuan（成本估算 ¥）。 Nice
GET /api/admin/llm-config 读取 LLM 配置（apiKey 脱敏显示）+ 调优参数（max_tokens / timeout / thinking_budget）。 Nice
POST /api/admin/llm-config 更新 LLM 配置：apiKey/baseUrl/model/updatedBy。未传字段保留原值。运行时可改无需重启。 Nice
安全设计亮点
LLM API Key 存储在 PocketBase secrets 表中（listRule/viewRule/createRule/updateRule/deleteRule 全为 null = 仅 admin 可访问），前端通过 /api/llm/stage1 和 /api/llm/stage2 代理调用，API Key 永远不暴露给前端。这是企业级的安全最佳实践。

1 三阶段引擎为空壳 · startGen() 仅 showToast P0

版本现状
v12 startGen() 只做了 showToast + goPage('review')，无三阶段判断逻辑。DecisionStream 组件存在但无数据驱动。
v11 runWikiJudgment(voices)（行 1469-1580+）：阶段 0 声音预处理 → 阶段 1 REQ 判断（LLM 优先 + 规则兜底）→ 阶段 2 TC 决策。每阶段推送 DecisionEvent 驱动卡片渲染。
PB 已有 LLM 代理端点：POST /api/llm/stage1 和 POST /api/llm/stage2。 后端已实现：system prompt 构造、模块词表注入、Wiki 状态感知、JSON 解析容错、llm_calls 日志自动写入。
v2 评估：工作量大幅降低
v1 报告认为需要"移植整个引擎"，但现在有了 PocketBase 后端代理，v12 只需：
① 前端写一个 async runWikiJudgment(voices) 循环调用 /api/llm/stage1 + /api/llm/stage2
② 将返回结果推送 DecisionEvent 渲染卡片
③ 调用 /api/wiki/persist 写回知识库

A Excel 列别名匹配 — 容错导入

用户导入的 Excel 列名千差万别：「编号」vs「ID」vs「序号」，v11 通过 VOICE_COLUMN_ALIASES 统一别名映射。

const VOICE_COLUMN_ALIASES = {
  id:      ['ID','id','声音ID','编号','序号'],
  model:   ['车型','型号','车辆','model'],
  content: ['声音内容','内容','投诉内容','原话','content'],
  channel: ['渠道','来源','channel'],
  status:  ['状态','status','处理状态']
};

B LLM 占位卡片 — 异步等待 UX

调用 LLM 前推送 processing 占位卡片，返回后替换。用户不会面对空白等待。

// 先推 "正在调用 LLM" 占位卡片
const placeholderId = pushDecisionEvent({
  stage: 1, voiceId: v.id, voiceContent: v.content,
  kind: 'processing',
  decisionText: '正在调用 DeepSeek…',
  _placeholder: true
});
const llmRes = await llmStage1Judgment(v);
_dstreamRemoveEvent(placeholderId);  // LLM 返回后移除占位
